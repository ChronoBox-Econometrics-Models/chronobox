# X-13 ARIMA-SEATS

!!! info "Quick Reference"
    - **Classe**: `chronobox.decomposition.x13_wrapper.X13Wrapper`
    - **Import**: `from chronobox.decomposition.x13_wrapper import X13Wrapper`
    - **Tipo**: Decomposicao model-based (aditiva ou multiplicativa)
    - **R equivalente**: `seasonal::seas()` / `stats::decompose()` com X-13
    - **Requisito**: executavel `x13as` instalado

---

## Overview

O **X-13ARIMA-SEATS** e o programa oficial do U.S. Census Bureau para ajuste
sazonal de series temporais. Ele e utilizado por agencias estatisticas no mundo
inteiro --- incluindo IBGE, Eurostat e Bank of England --- para produzir
indicadores dessazonalizados.

O programa combina dois frameworks:

- **X-11**: filtro de medias moveis iterativas para decomposicao
- **SEATS** (Signal Extraction in ARIMA Time Series): decomposicao baseada
  em modelo ARIMA

### Quando usar

- Ajuste sazonal de **indicadores oficiais** (PIB, inflacao, producao industrial)
- Necessidade de **compatibilidade** com ajustes de agencias estatisticas
- Analise de series com **efeitos de calendario** (dias uteis, feriados moveis)
- Quando se deseja decomposicao **model-based** com garantias teoricas

!!! warning "Requisito externo"
    O X-13ARIMA-SEATS requer o executavel `x13as` instalado no sistema.
    Download em: [census.gov/x13as](https://www.census.gov/data/software/x13as.html)

    Para a maioria dos usos, o **STL** e a alternativa recomendada pois nao
    requer dependencias externas.

## Formulacao Matematica

### Modelo ARIMA Regressivo (regARIMA)

O X-13 comeca ajustando um modelo **regARIMA** a serie:

$$
\phi(B)\,\Phi(B^s)\,(1-B)^d\,(1-B^s)^D\, y_t^* = \theta(B)\,\Theta(B^s)\, \varepsilon_t
$$

onde:

- $y_t^* = y_t - \sum_j \beta_j x_{jt}$ e a serie ajustada por regressores (calendario, outliers)
- $\phi(B)$ e $\Phi(B^s)$ sao polinomios AR regulares e sazonais
- $\theta(B)$ e $\Theta(B^s)$ sao polinomios MA regulares e sazonais
- $d$, $D$ sao ordens de diferenciacao
- $s$ e o periodo sazonal

### Decomposicao X-11

O metodo X-11 aplica **medias moveis simetricas** iterativamente:

1. **Estimacao inicial da tendencia**: media movel $2 \times 12$ (para dados mensais)
2. **Estimacao da sazonalidade**: media movel $3 \times 3$ nas subseries sazonais
3. **Ajuste sazonal preliminar**: $y_t^{SA} = y_t / S_t$ (multiplicativo)
4. **Refinamento**: repete os passos com filtros de Henderson
5. **Componentes finais**: tendencia, sazonal, irregular

### Decomposicao SEATS

O SEATS extrai os componentes como **sinais** do modelo ARIMA:

$$
y_t = T_t + S_t + I_t
$$

Cada componente tem seu proprio modelo ARIMA:

$$
T_t: \quad \phi_T(B)\,\Delta^d\, T_t = \theta_T(B)\,\varepsilon_{T,t}
$$

$$
S_t: \quad \phi_S(B)\,S(B)\, S_t = \theta_S(B)\,\varepsilon_{S,t}
$$

onde $S(B) = 1 + B + B^2 + \cdots + B^{s-1}$ e o operador de soma sazonal.

Os componentes sao extraidos via **filtro de Wiener-Kolmogorov**:

$$
\hat{T}_t = \frac{f_T(\omega)}{f_y(\omega)} \cdot y_t
$$

onde $f_T(\omega)$ e $f_y(\omega)$ sao as densidades espectrais da tendencia
e da serie observada, respectivamente.

!!! note "X-11 vs SEATS"
    - **X-11**: abordagem non-parametric baseada em filtros fixos. Mais estavel
      empiricamente, mas sem fundamentacao probabilistica formal
    - **SEATS**: abordagem model-based. Decomposicao otima sob o modelo ARIMA,
      mas depende da correta especificacao do modelo

## Quick Example

```python
from chronobox.decomposition.x13_wrapper import X13Wrapper
from chronobox.datasets import load_dataset

# Carregar producao industrial
y = load_dataset("producao_industrial")

# Inicializar wrapper (detecta x13as automaticamente)
x13 = X13Wrapper()

# Verificar disponibilidade
if x13.is_available:
    result = x13.seasonal_adjust(
        y.values,
        period=12,
        start_year=2002,
        start_month=1
    )
    print(result.summary())
else:
    print("X-13 nao disponivel. Instale o executavel x13as.")
```

## Guia Detalhado

### Construtor

```python
X13Wrapper(
    x13_path=None   # Caminho para o executavel x13as
)
```

| Parametro | Tipo | Default | Descricao |
|---|---|---|---|
| `x13_path` | `str \| None` | `None` | Caminho para `x13as`. Se `None`, busca automaticamente |

O wrapper busca o executavel nos seguintes locais:

1. `x13as`, `x13ashtml`, `x13as_ascii` no `PATH`
2. `/usr/local/bin/x13as`
3. `/usr/bin/x13as`
4. `~/x13as/x13as`

### Propriedade `is_available`

```python
x13 = X13Wrapper()
if x13.is_available:
    print("X-13 encontrado!")
else:
    print("X-13 nao disponivel")
```

### Metodo `seasonal_adjust()`

```python
result = x13.seasonal_adjust(
    endog,                  # Serie temporal (array-like)
    period=12,              # Periodo sazonal (12=mensal, 4=trimestral)
    start_year=2000,        # Ano inicial da serie
    start_month=1,          # Mes/trimestre inicial
    transform='auto'        # Transformacao: 'auto', 'log', 'none'
)
```

| Parametro | Tipo | Default | Descricao |
|---|---|---|---|
| `endog` | `array-like` | --- | Serie temporal |
| `period` | `int` | `12` | Periodo sazonal |
| `start_year` | `int` | `2000` | Ano inicial |
| `start_month` | `int` | `1` | Mes/trimestre inicial |
| `transform` | `str` | `'auto'` | Transformacao pre-ajuste |

**Opcoes de `transform`**:

| Valor | Descricao |
|---|---|
| `'auto'` | X-13 decide entre log e nenhuma (teste AIC) |
| `'log'` | Forca transformacao logaritmica |
| `'none'` | Sem transformacao |

### Resultado: `DecompositionResult`

O metodo retorna o mesmo `DecompositionResult` usado pelo STL:

| Atributo | Descricao |
|---|---|
| `observed` | Serie original |
| `trend` | Tendencia-ciclo (D12) |
| `seasonal` | Fatores sazonais (D10) |
| `remainder` | Componente irregular (D13) |
| `period` | Periodo sazonal |
| `model` | `'x13'` |

### Exemplo Completo com Plots

```python
import matplotlib.pyplot as plt
from chronobox.decomposition.x13_wrapper import X13Wrapper
from chronobox.datasets import load_dataset

y = load_dataset("producao_industrial")

x13 = X13Wrapper()
if not x13.is_available:
    raise RuntimeError("Instale o x13as para executar este exemplo")

result = x13.seasonal_adjust(
    y.values,
    period=12,
    start_year=2002,
    start_month=1,
    transform='auto'
)

# Plot dos componentes
fig, axes = plt.subplots(4, 1, figsize=(12, 8), sharex=True)

axes[0].plot(result.observed)
axes[0].set_ylabel("Observado")
axes[0].set_title("Decomposicao X-13 ARIMA-SEATS")

axes[1].plot(result.trend)
axes[1].set_ylabel("Tendencia")

axes[2].plot(result.seasonal)
axes[2].set_ylabel("Sazonal")

axes[3].plot(result.remainder)
axes[3].set_ylabel("Irregular")
axes[3].axhline(y=0, color='grey', linestyle='--', linewidth=0.8)

plt.tight_layout()
plt.show()
```

## Interpretacao

### Componentes de Saida

O X-13 produz diversas tabelas intermediarias. As principais, mapeadas para o
`DecompositionResult`:

| Tabela X-13 | Atributo | Descricao |
|---|---|---|
| D10 | `seasonal` | Fatores sazonais finais |
| D11 | --- | Serie dessazonalizada final |
| D12 | `trend` | Tendencia-ciclo final |
| D13 | `remainder` | Componente irregular final |

### Fatores Sazonais

Os fatores sazonais do X-11 sao **multiplicativos** por default:

- $S_t > 1$: o mes tem nivel **acima** da media sazonal
- $S_t < 1$: o mes tem nivel **abaixo** da media sazonal
- $S_t = 1$: sem efeito sazonal

```python
import numpy as np

# Fatores sazonais medios por mes
seasonal_by_month = result.seasonal.reshape(-1, 12)
monthly_means = np.nanmean(seasonal_by_month, axis=0)

months = ['Jan', 'Fev', 'Mar', 'Abr', 'Mai', 'Jun',
          'Jul', 'Ago', 'Set', 'Out', 'Nov', 'Dez']

for m, f in zip(months, monthly_means):
    print(f"{m}: {f:+.4f}")
```

### Qualidade do Ajuste

```python
import numpy as np

# Variancia explicada pelos componentes
total_var = np.nanvar(result.observed)
trend_var = np.nanvar(result.trend)
seasonal_var = np.nanvar(result.seasonal)
remainder_var = np.nanvar(result.remainder)

print(f"Variancia total:    {total_var:.2f}")
print(f"  Tendencia:        {trend_var:.2f} ({100*trend_var/total_var:.1f}%)")
print(f"  Sazonal:          {seasonal_var:.2f} ({100*seasonal_var/total_var:.1f}%)")
print(f"  Irregular:        {remainder_var:.2f} ({100*remainder_var/total_var:.1f}%)")
```

## Diagnosticos

### 1. Sazonalidade Residual

Verifique se o ajuste sazonal removeu toda a sazonalidade:

```python
from chronobox.tests_stat import ljung_box_test

# Testar autocorrelacao nos lags sazonais
lb = ljung_box_test(result.remainder, lags=24)
print(f"Ljung-Box p-value: {lb.pvalue:.4f}")
# p > 0.05 → sem sazonalidade residual
```

### 2. Estabilidade dos Fatores Sazonais

```python
import numpy as np

# Verificar se os fatores sazonais sao estaveis ao longo do tempo
seasonal_matrix = result.seasonal.reshape(-1, 12)
seasonal_std = np.nanstd(seasonal_matrix, axis=0)

print("Variabilidade dos fatores sazonais por mes:")
for i, std in enumerate(seasonal_std):
    status = "estavel" if std < 0.05 * np.nanmean(np.abs(seasonal_matrix[:, i])) else "variavel"
    print(f"  Mes {i+1:2d}: std = {std:.4f} ({status})")
```

### 3. Componente Irregular

```python
import numpy as np

# Detectar outliers no componente irregular
remainder = result.remainder[~np.isnan(result.remainder)]
threshold = 3 * np.std(remainder)
outliers = np.abs(remainder) > threshold

print(f"Outliers detectados (|R| > 3*sigma): {outliers.sum()}")
```

### Checklist de Diagnostico

| Verificacao | Como testar | Resultado esperado |
|---|---|---|
| Sem sazonalidade residual | Ljung-Box nos lags sazonais | $p > 0.05$ |
| Fatores sazonais estaveis | Desvio-padrao por periodo | Baixa variabilidade |
| Poucos outliers | $\|R_t\| > 3\sigma$ | $< 2\%$ das observacoes |
| Tendencia suave | Inspecao visual | Sem saltos abruptos |

## STL vs X-13: Quando Usar Cada

| Criterio | STL | X-13 ARIMA-SEATS |
|---|---|---|
| Dependencia externa | Nenhuma | Requer `x13as` |
| Tipo de decomposicao | Aditiva | Aditiva ou multiplicativa |
| Fundamentacao | Non-parametric | Model-based (ARIMA) |
| Efeitos de calendario | Nao | Sim |
| Outlier detection | Via pesos de robustez | Automatico (AO, LS, TC) |
| Uso oficial | Nao | Padrao de agencias |
| Facilidade de uso | Alta | Moderada |
| Recomendacao | Uso geral, exploratoria | Indicadores oficiais |

!!! tip "Recomendacao pratica"
    Para a maioria das analises, use **STL**. Reserve o X-13 para quando
    precisar de compatibilidade com ajustes de agencias estatisticas ou
    quando efeitos de calendario forem relevantes.

## Equivalentes R

=== "chronobox (Python)"

    ```python
    from chronobox.decomposition.x13_wrapper import X13Wrapper

    x13 = X13Wrapper()
    result = x13.seasonal_adjust(
        y,
        period=12,
        start_year=2000,
        start_month=1,
        transform='auto'
    )

    print(result.summary())
    ```

=== "seasonal (R)"

    ```r
    library(seasonal)

    # Ajuste sazonal com X-13
    fit <- seas(AirPassengers)

    summary(fit)
    plot(fit)

    # Serie dessazonalizada
    sa <- final(fit)

    # Componentes
    trend <- trend(fit)
    seasonal <- seasonal(fit)
    irregular <- irregular(fit)
    ```

=== "x13binary (R)"

    ```r
    library(seasonal)
    library(x13binary)

    # Verifica e usa o binario bundled
    checkX13()

    fit <- seas(
      AirPassengers,
      x11 = "",           # Usar X-11 (default e SEATS)
      transform.function = "auto"
    )

    summary(fit)
    ```

**Mapeamento de parametros**:

| chronobox | seasonal::seas (R) | Descricao |
|---|---|---|
| `period=12` | Definido no `ts()` | Periodo sazonal |
| `transform='auto'` | `transform.function="auto"` | Transformacao |
| `transform='log'` | `transform.function="log"` | Log transform |
| `transform='none'` | `transform.function="none"` | Sem transformacao |
| `result.trend` | `trend(fit)` | Tendencia-ciclo |
| `result.seasonal` | `seasonal(fit)` | Fatores sazonais |
| `result.remainder` | `irregular(fit)` | Componente irregular |

!!! note "Pacote seasonal vs chronobox"
    O pacote `seasonal` do R oferece uma interface muito mais rica para o X-13,
    incluindo diagnosticos automaticos, graficos especializados e controle
    fino de todas as specs. O `X13Wrapper` do chronobox fornece uma interface
    simplificada focada na extracao dos componentes principais.

## Instalacao do X-13

### Linux

```bash
# Download do binario
wget https://www2.census.gov/software/x-13arima-seats/x13as/unix-linux/program/x13as_ascii-v1-1-b60.tar.gz

# Extrair
tar -xzf x13as_ascii-v1-1-b60.tar.gz

# Mover para o PATH
sudo mv x13as /usr/local/bin/

# Verificar
x13as
```

### macOS

```bash
# Via Homebrew (se disponivel)
brew install x13as

# Ou download manual do Census Bureau
```

### Windows

1. Baixe o executavel em [census.gov/x13as](https://www.census.gov/data/software/x13as.html)
2. Extraia para `C:\x13as\`
3. Adicione `C:\x13as\` ao `PATH` ou passe o caminho:

```python
x13 = X13Wrapper(x13_path=r"C:\x13as\x13as.exe")
```
